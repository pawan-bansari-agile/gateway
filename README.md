## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Installation

```bash
$ npm install
```

## Build graphql types

```bash
$ npm run build:types
```

## Running the app

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## GraphQl playground will be exposed at the below endpoing

http://localhost:3000/graphql

## How the sytem works

This is an GraphQl gateway which is exposing functionalities of the Books and Users microservices. THe book service handles the CRUD operations on a book and the User service handles the CRUD on a user. The gateway uses TCP protocol while the system is integrated using the gRPC protocol.

For better understanding of the gRPC services you can refer the grpc.interface.ts which defines the basic structure of our services.

Each service has its own database. The user database is shared with the user service for user CRUD operations and the gateway for authentication only.

The gateway handles user authentication to reduce redundant calls to the user service to check user authorisation.

As this is a basic demo setup, I've kept the env variable static and used directly instead of making an .env to keep things simple.

For example queries and mutations you can refer to the file payloads in this repo.

To start the services just execute the below commands in the terminal:-

npm i && npm run start:dev

book service will run on port 50051
user service will run on port 50052
